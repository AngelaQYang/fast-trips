static find_trip_based_path(FT, passenger, path, hyperpath, trace)[source]¶

Perform trip-based path search.

Will do so either backwards (destination to origin) if Path.direction is Path.DIR_OUTBOUND or forwards (origin to destination) if Path.direction is Path.DIR_INBOUND.

Returns (path cost, return_states).

Parameters:
  • FT (a FastTrips instance) – fasttrips data
  • passenger (a Passenger instance) – the passenger whose path we’re finding
  • path (a Path instance) – the path to fill in
  • hyperpath (boolean) – pass True to use a stochastic hyperpath-finding algorithm, otherwise a deterministic shortest path search algorithm will be use.
  • trace (boolean) – pass True if this path should be traced to the debug log

static setup_passengers(FT, output_dir, iteration)[source]¶

Converts assignment results (which is stored in each Passenger Path, in the Path.states) into a single pandas.DataFrame. Each row represents a link in the passenger’s path. The returned pandas.DataFrame has the following columns:

column name column type description
passenger_id int64 the Passenger.passenger_id
path_id int64 a sequential integer ID unique to each Path instance
pathdir int64 the Path.direction
pathmode int64 the Path.mode
linkmode object the mode of the link, one of Path.STATE_MODE_ACCESS, Path.STATE_MODE_EGRESS, Path.STATE_MODE_TRANSFER or Path.STATE_MODE_TRIP. Paths will always start with access, followed by trips with transfers in between, and ending in an egress following the last trip.
trip_id float64 the Trip.trip_id for trip links. Set to numpy.nan for non-trip links.
A_id float64 the Stop.stop_id at the start of the link, or a TAZ.taz_id for access links
B_id float64 the Stop.stop_id at the end of the link, or a TAZ.taz_id for access links
A_seq, int64 the sequence number for the stop at the start of the link, or -1 for access links
B_seq, int64 the sequence number for the stop at the start of the link, or -1 for access links
A_time datetime64[ns] the time the passenger arrives at A_id
B_time datetime64[ns] the time the passenger arrives at B_id
linktime timedelta64[ns] the time spent on the link
cost float64 the cost of the entire path

Additionally, this method writes out the dataframe to a csv at Assignment.PASSENGERS_CSV in the given output_dir and labeled with the given iteration.

BUMP_ONE_AT_A_TIME = True¶

Simulation: bump one stop at a time (slower, more accurate)

When addressing capacity constraints in simulation, we look at all the (trip, stop)-pairs where the boards are not allowed since vehicle is over capacity. The faster way to address this is to bump all of those passengers, which means we call the assigned path bad and try to reassign.

However, this could over-bump passengers, because if a passenger rides multiple crowded vehicles, then bumping her frees up space on other vehicles and so some other bumping may not be necessary. Thus, the more accurate (but slower) method is to bump passengers from each (trip,stop) at a time, in order of the full vehicle arrival time, and then recalculate loads, and iterate until we have no capacity issues.

Route choice configuration: Dispersion parameter in the logit function. Higher values result in less stochasticity. Must be nonnegative. If unknown use a value between 0.5 and 1

ITERATION_FLAG = 1¶

Configuration: Maximum number of iterations to remove capacity violations. When the transit system is not crowded or when capacity constraint is relaxed the model will terminate after the first iteration

NUMBER_OF_PROCESSES = 14¶

Number of processes to use for path finding (via multiprocessing) Set to 1 to run everything in this process Set to less than 1 to use the result of multiprocessing.cpu_count() Set to positive integer greater than 1 to set a fixed number of processes
