static find_trip_based_hyperpath(FT, path, trace)¶

Perform backwards (destination to origin) trip-based hyperpath search. Will do so either backwards (destination to origin) if Path.direction is Path.DIR_OUTBOUND or forwards (origin to destination) if Path.direction is Path.DIR_INBOUND.

Updates the path information in the given path and returns the number of label iterations required.

Parameters:
  • FT (a FastTrips instance) – fasttrips data
  • path (a Path instance) – the path to fill in
  • trace (boolean) – pass True if this path should be traced to the debug log

CAPACITY_CONSTRAINT = False¶

Route choice configuration: Use vehicle capacity constraints

CREATE_SKIMS = False¶

Configuration: Create skims flag. This is specific to the travel demand models (not working in this version)

DISPERSION_PARAMETER = 1.0¶

Route choice configuration: Dispersion parameter in the logit function. Higher values result in less stochasticity. Must be nonnegative. If unknown use a value between 0.5 and 1

ITERATION_FLAG = 1¶

Configuration: Maximum number of iterations to remove capacity violations. When the transit system is not crowded or when capacity constraint is relaxed the model will terminate after the first iteration

MAX_HYPERPATH_ASSIGN_ATTEMPTS = 1001¶

Maximum number of times to call choose_path_from_hyperpath_states()

OUTPUT_PASSENGER_TRAJECTORIES = True¶

Configuration: Passenger trajectory output flag. Passengers’ path and time will be reported if this flag is on. Note that the simulation flag should be on for passengers’ time.

PATH_TIME_WINDOW = datetime.timedelta(0, 1800)¶

Configuration: Path time-window. This is the time in which the paths are generated. E.g. with a typical 30 min window, any path within 30 min of the departure time will be checked.

RAND_FILE = None¶

The file handle for Assignment.RAND_FILENAME

RAND_FILENAME = '../FAST-TrIPs-1/rand/rand.txt'¶

Temporary rand.txt location for Assignment.test_rand().

RAND_MAX = 0¶

Maximum random number returned by Assignment.test_rand()

SIMULATION_FLAG = True¶

Configuration: Simulation flag. It should be on for iterative assignment. In a one shot assignment with simulation flag off, the passengers are assigned to paths but are not loaded to the network.

SKIM_END_TIME = 600¶

Configuration: End of the time period for which the skim is required (minutes from start of day)

SKIM_START_TIME = 300¶

Configuration: Beginning of the time period for which the skim is required. (in minutes from start of day)
